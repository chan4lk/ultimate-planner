# User Auth & Profile Service

## Product Requirement Document (PRD)

**Purpose:**  
Manage user authentication, authorization, and profile data.

**Core Requirements:**  
- Support OAuth2/social login (Microsoft, Google, Notion, etc.).  
- Store user profile and integration tokens securely.  
- Expose API for user profile management.  
- Enforce access control for all APIs.

## Rule File (Acceptance Criteria)

- [ ] Users can log in with supported providers.
- [ ] Tokens are stored securely and refreshed as needed.
- [ ] User profile data is available via API.
- [ ] All APIs enforce proper access control.
- [ ] Logs all auth operations.
